## 一般

### このツールは何のためのものですか？

EasingVisualizerは**イージング関数**の視覚的比較ツールで、以下のために特別に設計されています：

- **Beat Saberモッディング** - マップやMod用のアニメーションカーブを可視化
- **ScriptMapperユーザー** - イージング関数がゲーム内でどのように見えるかを正確にプレビュー
- **アニメーションワークフロー** - 異なるイージング動作を理解・比較
- **学習** - 数学的イージング関数を実際に動作させて確認

### 誰が作りましたか？

EasingVisualizerは、ScriptMapperやその他のアニメーションツールで使用されるイージング関数を可視化するために、Beat Saberマッピングコミュニティ向けに作成されました。

## ScriptMapper

### ScriptMapperとは？

ScriptMapperは、Beat Saberでカスタムマップに高度なアニメーションとエフェクトを可能にするスクリプティングツールです。時間経過に伴う値の変化を制御するためにイージング関数を使用します。

詳細は[[ScriptMapper-Mode.ja|ScriptMapperモード]]を参照してください。

### ScriptMapperで使えるイージング関数は？

EasingVisualizerのすべての標準イージング関数がScriptMapperと互換性があります：

- **Quad, Cubic, Quart, Quint** - 多項式カーブ
- **Sine, Expo, Circ** - スムーズカーブ
- **Back, Elastic, Bounce** - 特殊エフェクト
- **Linear** - イージングなし

それぞれScriptMapperで `easeIn`、`easeOut`、または `easeInOut` として使用できます。

### ScriptMapperモードの使い方は？

1. 詳細設定で **ScriptMapperモード** を有効化
2. パネルにScriptMapper互換の関数名が表示されます
3. 一般的な設定用にScriptMapperプリセットセレクターを使用

## 共有と保存

### 設定を共有するには？

1. パネルを希望通りにセットアップ
2. ツールバーの **Share** ボタンをクリック
3. URLがクリップボードにコピーされます
4. 誰にでもURLを送信 - 受信者はあなたの正確な設定を見られます

詳細は[[URL-Sharing|URL共有]]を参照してください。

### 設定は保存されますか？

はい！設定はブラウザの **localStorage** に自動的に保存されます：

- パネル設定
- 速度とガンマ設定
- カスタムプリセット

設定はブラウザセッション間で保持されますが、デバイス固有です。

### プリセットを保存するには？

1. パネルを設定
2. プリセットマネージャーで **Save Preset** をクリック
3. プリセット名を入力
4. プリセットドロップダウンからいつでもアクセス可能

詳細は[[Preset-System|プリセットシステム]]を参照してください。

## 技術

### 対応ブラウザは？

| ブラウザ | サポート |
|---------|---------|
| Chrome 90+ | ✅ 完全対応 |
| Firefox 88+ | ✅ 完全対応 |
| Safari 14+ | ✅ 完全対応 |
| Edge 90+ | ✅ 完全対応 |
| Opera 76+ | ✅ 完全対応 |

古いブラウザでも動作する可能性がありますが、公式にはサポートされていません。

### モバイルで動作しますか？

はい！EasingVisualizerは完全にレスポンシブで、以下を備えています：

- タッチ最適化コントロール
- モバイルコントロールシート
- レスポンシブパネルグリッド

詳細は[[Mobile-Support|モバイルサポート]]を参照してください。

### システム要件は？

- **WebGLサポート** - カメラプレビューに必要
- **モダンブラウザ** - 上記のブラウザサポートを参照
- **JavaScript有効** - 機能に必要

### カメラプレビューが遅いのはなぜですか？

カメラプレビューは3DレンダリングにThree.jsを使用しています。遅い場合：

1. 他のブラウザタブを閉じる
2. パネル数を減らす
3. シンプルなプレビュータイプ（glow、value）に切り替え
4. ブラウザでハードウェアアクセラレーションが有効か確認

## 機能

### 異なるプレビュータイプは何を表示しますか？

| プレビュー | 表示内容 |
|-----------|---------|
| **Glow** | 不透明度/強度の変化 |
| **Graph** | 数学的カーブの可視化 |
| **Camera** | 3Dカメラ移動パス |
| **Value** | 数値表示 |

詳細な説明は[[Preview-Types.ja|プレビュータイプ]]を参照してください。

### easein、easeout、easebothの違いは？

- **easein** - ゆっくり始まり、速く終わる
- **easeout** - 速く始まり、ゆっくり終わる
- **easeboth** (easeInOut) - 両端がゆっくり、中間が速い

視覚的な比較は[[Easing-Functions.ja|イージング関数]]を参照してください。

### カスタムイージング関数を追加できますか？

開発者向けには、はい！コードベースへの新しいイージング関数の追加方法については[[Contributing|コントリビューション]]を参照してください。

## トラブルシューティング

### アプリが読み込まれません

1. ブラウザのキャッシュをクリア
2. ブラウザ拡張機能を無効化
3. 別のブラウザを試す
4. ブラウザコンソールでエラーを確認

### 設定が消えました

設定はlocalStorageに保存されています。以下の場合にクリアされる可能性があります：

- ブラウザデータをクリアした
- プライベート/シークレットモードを使用している
- ブラウザのストレージ制限に達した
- ブラウザ拡張機能がlocalStorageをクリアしている

### アニメーションがカクつきます

1. パネル数を減らす（4-6から開始）
2. シンプルなプレビュータイプを使用
3. 他のタブ/アプリケーションを閉じる
4. ハードウェアアクセラレーションを有効化
